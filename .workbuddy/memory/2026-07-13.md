# 2026-07-13

## 修复：Flask 后端「能跑通但没数据」

### 根因
`config.py:34` 写成了 `BASE_DIR = Path(__file__).resolve().parents[1]`，
但 `config.py` 位于项目根 `D:\AI2.0\config.py`，其父目录（parents[0]）才是
项目根。结果 `config.OUTPUT_DIR = D:\output`（错误），而真实数据在 `D:\AI2.0\output`。

读路径（`/api/report`、`/api/meta`、`/api/export`、sources 配置）都经由
`config.OUTPUT_DIR`（经 `issue_paths` 间接导入），找不到文件 →
`load_latest_report_dict()` 返回 None → `/api/report` 返回空 `[]` → 前端无数据。
写路径（`report_builder.py`、`web_server.py`）各自算对了路径，所以数据正常落盘，
只是读不到。

### 改动
1. `config.py:34` `parents[1]` → `parents[0]`（OUTPUT_DIR/NEWSLETTER_CONFIG_PATH 均回归正确）。
2. `app/services/report_reader.py` `to_frontend_articles`：文章图片 URL 由
   `/{issue_dir}/{img}` 改为 `/output/{issue_dir}/{img}`，匹配 Flask 的
   `/output/<path>` 静态路由（否则图片 404）。
3. `web_server.py` `api_meta`：cover_image 去掉多余的 `weekly-{date}/` 前缀，
   只返回 `images/cover.png`，由前端 `loadMeta` 拼成
   `/output/weekly-{date}/images/cover.png`（原逻辑双重前缀导致 404）。

### 验证
venv（python-dotenv + flask）启动 `web_app.py` 实测：
`/api/report` 200 返回 30 条；文章图与封面均 200；`/api/archive` 200。

- 用户要求文章板块（时间线卡片）不显示配图：删除 `static/js/render.js` 中
  `renderArticle` 的图片 `<img>` 块（保留头部封面 cover 不变）。前端不再渲染

  `overview.cover_image` 存为 `weekly-<date>/images/cover.png`，经 Flask
  `/output/...` 路由对外提供（实测 200）。仓库内仅此一处 cover.png。

## 三次改动：用生成的 cover.png 替换固定 banner
- 用户的「固定图片」是 `static/images/banner.png`，写在 `templates/index.html`
  的 `header-banner` 里作为页眉固定横幅（一直显示）。已删除该 `<img>` 引用，
  页眉改由 `#coverImage`（AI 生成的 `output/weekly-<date>/images/cover.png`）承担。
- 顺带修掉 `static/js/app.js` 的 `renderCover()` 封面路径 bug：原先拼成
  `/weekly-<date>/images/cover.png`（缺 `/output/` 前缀，404）。改为统一拼成
  `/output/weekly-{date}/<rel>`，与 Flask `/output/<path>` 路由匹配（实测 200）。
- 说明：`banner.png` 文件本身未删除（保留在 static/images/，如要清理可手动删）。
  `loadMeta` 路径此前已修正为 `/output/weekly-{date}/images/cover.png`，与之一致。
